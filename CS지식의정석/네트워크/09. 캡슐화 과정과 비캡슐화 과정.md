# 캡슐화 과정

1. 어플리케이션
  - 데이터

2. 전송
  - tcp헤더 + 데이터

3. 인터넷
  - ip(L3) 헤더 + tcp헤더 + 데이터

4. 링크
  - 프레임 헤더 + ip(L3) 헤더 + tcp헤더 + 데이터

# 비 캡슐화 과정

1. 링크
  - 프레임 헤더 + ip(L3) 헤더 + tcp헤더 + 데이터

2. 인터넷
  - ip(L3) 헤더 + tcp헤더 + 데이터

3. 전송
  - tcp헤더 + 데이터

4. 애플리케이션
  - 데이터

# PDU

- 애플리케이션 계층 : 메세지
- 전송 계층 : 세크먼트(TCP), 데이터 그램(UDP)
- 인터넷 계층 : 패킷
- 링크 계층 : 프레임(데이터 링크 계층), 비트(물리 계층)

# 정리 :

- 애플리케이션 계층 -> 전송계층으로 전달
- 세그먼트 또는 데이터그램화 + TCP 헤더 붙여짐
- 전송계층 -> 인터넷 계층 전달
- IP(L3)헤더가 붙여짐, 패킷화
- 인터넷 계층 -> 링크 계층
- 프레임 페더, 프레임 트레일러 추가 프레임화
- request 캡슐화 과정을 마침
- 그리고 다시 반대로 링크 계층 -> 어플리케이션 계층 으로 비 캡슐화 를 거처 서버 전송
